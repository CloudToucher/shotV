# 项目设计文档

## 1. 项目定位

本项目定义为：

`单机离线的 2D 几何风撤离式 roguelite`

核心目标不是做一个依赖海量素材堆内容的搜打撤，也不是做 PvP 竞技，而是做一个：

- 有高压战斗反馈
- 有大地图中的多路线选择
- 有高风险深入与撤离决策
- 有局内搜集与局外构筑闭环
- 可通过程序化生成持续提供变化

项目当前以 `Vue + PixiJS` 为基础，优先完成完整可玩的 MVP 闭环，再考虑扩展怪物、地图、区域、掉落池和玩法广度。

## 2. 核心体验

玩家每一局的体验应该围绕以下决策展开：

1. 从局外基地选择当前构筑、带入资源和合同
2. 进入一张足够大的战区地图，自主决定探索路径
3. 在外围区、高风险区、深层高价值区之间做风险取舍
4. 搜集资源、钥匙卡、装备、模块和任务目标
5. 在弹药、弹种、血量、护甲耐久、武器耐久、背包空间和怪物注意力都有限制的情况下决定继续深入还是撤离
6. 撤离成功后将部分战利品带回基地，形成下一局的新构筑与成长

玩家反复开局的动机，不是单纯分数，也不是纯金币，而是：

- 这一局可能带回改变玩法的新模块或新装备
- 这一次深入高风险区可能值回票价，也可能直接翻车
- 这次撤离结果会影响下一局的配装、技能和路线

## 3. 设计原则

- 用 `系统组合` 替代 `素材堆量`
- 用 `风险决策` 替代 `纯刷数值`
- 用 `构筑变化` 替代 `简单成长`
- 用 `程序化规则` 替代 `大批手工内容`
- 用 `几何视觉 + 强反馈` 建立识别度
- 先打通 MVP 全链路，再扩展广度

## 4. 视觉与美术方向

### 4.1 风格关键词

- 极简
- 几何
- 高对比
- 现代战术终端
- 冷色安全区 / 暖色危险区 / 高亮价值区

### 4.2 美术策略

由于美术资源有限，视觉设计必须天然适合程序化生成。

建议采用：

- 角色、敌人、枪械、掉落物全部使用几何体和程序化图形
- 用轮廓、前向箭头、内核亮点区分单位类型
- 用颜色、描边、脉冲、标记表达稀有度和危险等级
- 用地面网格、警戒线、警示条、灯带、扫描圈、终端投影构建场景层次
- 用少量图标和 UI 符号承担语义，而不是依赖复杂贴图

### 4.3 程序化地图视觉构成

地图不依赖大量独立素材，而由以下元素组合：

- 地板样式
- 墙体模块
- 障碍体块
- 门禁区域
- 箱体与终端
- 危险源
- 灯光与警示图形
- 区域色相和雾效

这样可以用少量资源生成大量视觉差异。

## 5. 技术栈与职责划分

### 5.1 技术选型

- `Vue`：局外基地、仓库、工坊、研究、成就、存档管理、菜单 UI
- `PixiJS`：战斗、地图渲染、敌人、掉落、弹幕、特效、局内交互
- 存档：本地 `IndexedDB`

### 5.2 数据驱动

以下系统必须尽量数据驱动：

- 武器定义
- 护甲定义
- 消耗品定义
- 掉落物定义
- 模块定义
- 技能定义
- 成就定义
- 区域定义
- 敌人词缀定义
- 合同定义
- 资源定义

项目后续扩展广度时，应主要依赖新增数据表，而不是大量改动核心逻辑。

## 6. MVP 范围

MVP 的目标不是做完整大作，而是打通以下闭环：

- 局外基地
- 进入战区
- 大地图探索
- 战斗与搜刮
- 高风险区域
- 钥匙卡开启高价值区
- 背包管理
- 武器与护甲耐久
- 基础甲弹对抗
- 基础怪物注意力与噪声系统
- 消耗品使用
- 撤离
- 局外成长
- 离线基础收益

### 6.1 MVP 必做内容

- 一张足够大的程序化战区地图
- 至少 3 种基础武器
- 至少 2 套护甲
- 至少 4 种消耗品
- 基础背包网格系统
- 至少 2 类基础弹种
- 至少 3 类普通敌人 + 1 名 mini boss
- 高风险区和高价值区
- 钥匙卡获取与门禁机制
- 局外武器升级与技能成长
- 基础怪物注意力系统与消音器模组
- 基础成就解锁与掉落扩展
- 离线挂机收益系统

### 6.2 MVP 不做内容

- 联机
- 交易市场
- 复杂剧情演出
- 大量地图主题
- 大量枪械种类
- 大量 boss
- 复杂 NPC 系统
- 复杂制造树

## 7. 局外系统

### 7.1 基地主场景

局外主场景为 `前哨中继站`，由以下功能模块组成：

- `行动终端`：选择战区、合同、难度修正、带入物资
- `武器工坊`：枪械升级、武器维修、模组安装
- `护甲维护台`：护甲修复、护甲强化
- `仓库`：资源管理、装备管理、背包整理
- `研究实验室`：技能成长、掉落池解锁、特殊科技
- `无人机平台`：离线收益、自动回收、挂机派遣
- `成就面板`：成就进度和对应解锁内容

### 7.2 局外成长目标

局外成长不应主要依赖线性数值堆叠，而要优先提供：

- 新武器升级节点
- 新模块槽位
- 新消耗品配方
- 新技能节点
- 新高风险合同
- 新掉落类型
- 新区域事件

### 7.3 长期成长层级

建议拆成三层：

- `基础保障层`：背包容量、仓库容量、基础维修效率、离线收益效率
- `战斗构筑层`：武器升级、护甲特性、技能专精
- `内容解锁层`：新掉落池、新区域变体、新敌人词缀、新深层门禁规则

### 7.4 基建材料与资源层级

局外资源不只用于升级角色，还要明确承担 `基建` 和 `后勤` 功能。

建议 MVP 的资源层级如下：

- `废料`：基础维修、普通弹药补给、低级道具制造
- `零件`：武器维护、消音器和模块制造、局内应急修理工具
- `结构合金`：基地模块升级、背包扩容、仓库扩容
- `能源核心`：高风险区设备供能、深层设施解锁、离线平台升级
- `研究样本`：技能树、掉落池、特殊科技解锁

原则如下：

- 基础资源可通过普通探索和离线收益获得
- 基建材料以 `结构合金` 和 `能源核心` 为主，必须依赖主动游玩
- 高级成长不允许完全由挂机覆盖

## 8. 局内战区与地图设计

### 8.1 地图方向

地图必须足够大，提供真实路线选择，而不是单纯一张小竞技场。

但地图不应做成无边无际的开放世界，而应采用：

`大区域地图 + 分区节点 + 程序化房间群`

### 8.2 地图结构

每局地图由多个区块拼接而成：

- `入口区`
- `外围探索区`
- `资源密集区`
- `高风险区`
- `深层高价值区`
- `撤离点区`

每个区块都带有：

- 风险等级
- 掉落权重
- 事件标签
- 环境危害
- 特殊敌人池
- 门禁关系

### 8.3 地图大小建议

MVP 地图建议做成：

- 单局 10 到 20 分钟可完成
- 玩家从入口到最深层不走回头路也需要明显推进时间
- 同时存在多条外围路径、至少 2 条高风险深入路线
- 玩家无法在一局内轻松搜完整图

这点非常重要。地图的“够大”，不是绝对面积，而是：

- 选择足够多
- 路线足够不同
- 撤离成本足够真实

### 8.4 程序化生成方式

建议按区块生成，而不是纯随机散点。

每张图由：

- 若干大区块模板
- 每个区块内的若干子房间模板
- 障碍、终端、箱体、危险源、巡逻点、精英刷新点的参数化布局

这样可以保证：

- 地图可读
- 路径有意图
- 风险分布可控

## 9. 区域层级设计

### 9.1 外围区

- 风险低
- 基础资源多
- 普通敌人为主
- 适合补给和初期装备成型

### 9.2 高风险区

- 精英怪更多
- 环境危害更强
- 弹药和耐久消耗更高
- 未消音射击更容易吸引额外巡逻与增援
- 更高概率掉落钥匙卡、稀有模块和强化材料

### 9.3 深层高价值区

进入条件：

- 使用钥匙卡
- 或完成特定终端、击杀精英、收集钥匙卡碎片

特点：

- 高密度战斗
- mini boss 或强化守卫
- 高价值掉落
- 更高死亡风险

这里是单局中最具吸引力的区域，必须承担“值得赌一把”的价值。

## 10. 撤离式战斗系统

### 10.1 单局流程

1. 基地准备
2. 进入战区
3. 搜集资源与击杀敌人
4. 获取钥匙卡或碎片
5. 决定是否进入高风险区
6. 决定是否开启高价值区
7. 带着战利品前往撤离点
8. 成功撤离或死亡结算

### 10.2 撤离压力

为了让撤离决策成立，局内必须存在持续压力：

- 敌人强度随时间上升
- 高风险区会消耗更多弹药和耐久
- 护甲和武器状态持续恶化
- 背包空间有限
- 噪声和战斗行为会持续提高周边敌人的警觉度
- 撤离点有开启条件、距离成本或延迟

### 10.3 死亡与损失

死亡后应有真实损失，但不能完全清零。

建议：

- 当前局未保险的战利品全部丢失
- 基础资源保留少量结算
- 任务进度和成就进度保留
- 研究类长期成长保留

这样玩家会在“冒险”和“保底”之间做决策。

### 10.4 怪物注意力与噪声系统

MVP 需要一套简化但明确的注意力系统。

敌人状态建议为：

- `巡逻`
- `警觉`
- `搜寻`
- `战斗`

注意力的主要来源：

- 未消音枪声
- 爆炸
- 冲刺碰撞
- 终端启动和门禁开启
- 敌人之间的警报传播

系统目标：

- 让玩家可以通过行动方式控制战斗规模
- 让高风险区的路线选择不只是看敌人数值，还要看噪声成本
- 让消音器和诱饵类道具拥有明确战略价值

## 11. 背包系统

### 11.1 方向

优先采用 `网格背包系统`。

因为它天然能制造真实取舍：

- 不是拿到就捡
- 不是只比数值
- 玩家需要判断体积、价值、用途、风险

### 11.2 MVP 背包结构

建议初始背包为固定网格，例如：

- 主背包：`8 x 6`
- 快捷栏：4 格
- 装备栏：武器主手、副手、护甲、战斗道具

### 11.3 物品尺寸

物品按尺寸占格（大致）：

- 弹药：`1 x 1`
- 医疗包、小型材料：`1 x 2`
- 战斗道具：`2 x 1`
- 武器模块：`2 x 2`
- 主武器：`2 x 3`
- 护甲部件或大型战利品：`3 x 2`

### 11.4 物品类别

- 资源
- 基建材料
- 钥匙卡
- 弹药
- 消耗品
- 武器
- 模块
- 任务物
- 护甲材料

### 11.5 背包体验目标

背包不是为了复杂而复杂，而是为了产生这些问题：

- 我要带更多补给，还是留空间装战利品
- 我要拿重型高价值物品，还是拿更多小件资源
- 我要现在就撤，还是继续深入赌更好的掉落

### 11.6 网格实现要求

MVP 只做以下必要能力：

- 拖拽放置
- 自动整理
- 旋转物品
- 快速拾取
- 丢弃
- 快捷栏绑定

不做：

- 复杂容器嵌套
- 物品重量模拟
- 超复杂堆叠规则

## 12. 武器、护甲与耐久系统

### 12.1 武器系统

MVP 保留 3 类武器：

- 机枪：持续输出、破盾、清杂
- 榴弹：范围清场、区域压制
- 狙击：高精度、高单发、高弱点收益

### 12.2 换弹系统

局内需要真实换弹，形成战斗节奏。

枪械必须包含：

- 弹匣容量
- 当前弹匣
- 备用弹药
- 换弹时间
- 射击后坐反馈

### 12.3 子弹与弹种系统

MVP 不需要复杂到按现实口径拆分，但需要有基础弹种差异。

建议做以下弹种：

- `标准弹`：泛用、便宜、适合普通敌人
- `穿甲弹`：对重甲目标和高护甲单位更有效，但资源成本更高
- `爆破弹药`：主要服务榴弹或特殊武器，稀有且占背包空间

设计原则：

- 弹药必须占据背包格子
- 特殊弹药的带入会压缩战利品空间
- 高级弹药不应该成为无限常驻，而是局内的战术选择

### 12.4 甲弹对抗

甲弹对抗必须用简化规则实现，不追求复杂拟真。

建议采用：

- 敌人和护甲都有 `护甲等级`
- 子弹和武器都有 `穿透等级`
- 穿透等级低于目标护甲等级时，伤害更多被护甲吸收
- 穿透等级达到或高于目标护甲等级时，更多伤害直接作用于生命或核心部位

这样可以自然形成：

- 机枪 + 标准弹：清杂和打轻甲
- 狙击或穿甲弹：处理高甲单位和弱点窗口
- 榴弹：绕过部分正面对抗，打范围和逼位

### 12.5 武器耐久

武器耐久建议采用轻中度损耗：

- 持续射击、受环境干扰、死亡结算会损耗耐久
- 武器耐久越低，稳定性、射速或故障率受影响
- 耐久归零不直接永久报废，但会显著影响使用体验

局外可维修，局内只能进行有限应急维护。

### 12.6 护甲系统

护甲应作为独立层存在，而不是单纯生命值放大。

护甲应具备：

- 耐久值
- 护甲等级
- 吸收比例
- 抗性倾向
- 特殊词缀

举例：

- 轻甲：机动更好，耐久较低
- 中甲：平衡
- 重甲：抗性更高，但影响操作

### 12.7 护甲耐久

- 受到攻击时优先损耗护甲耐久
- 护甲损坏后保护能力下降
- 局内可使用有限修复包恢复部分耐久
- 局外使用资源进行完整修复

### 12.8 消音器与隐蔽战斗

消音器在本项目中不是纯数值配件，而是 `路线与风险控制工具`。

效果建议：

- 显著降低枪声传播范围
- 降低首次交战时引来的敌人数量
- 提高对巡逻敌人的先手击杀稳定性

代价建议：

- 略微降低伤害或子弹速度
- 增加武器耐久损耗
- 占用模组槽位

这样消音器才会真实影响玩家是否能低成本通过高风险区外围。

## 13. 枪械升级与模组

### 13.1 枪械升级目标

枪械升级应该优先改变玩法，而不是只提高数字。

升级分三层：

- `基础平台升级`：精度、换弹、弹匣、稳定性
- `机制升级`：穿透、爆裂、分裂、标记、电弧、弱点增伤
- `模组插槽`：允许挂载局内带回的特殊模块

### 13.2 模组设计原则

模组最好采用：

`触发条件 + 效果 + 代价`

示例：

- 冲刺后下一枪爆炸
- 弱点击杀后自动装填一部分弹药
- 护甲破损时获得短暂伤害加成
- 高风险区中伤害提升但武器耐久损耗增加

## 14. 战斗道具与消耗资源

### 14.1 道具类别

- 医疗类
- 护甲修复类
- 控场类
- 爆发类
- 位移/脱困类

### 14.2 MVP 道具建议

- 小型治疗针
- 护甲修复包
- 诱饵装置
- EMP 雷
- 爆破包

### 14.3 消耗资源逻辑

道具需要局外资源制造或补充。

这样玩家在出发前会考虑：

- 本局投入多少成本
- 高风险深入是否值得带更多补给
- 是保守打法还是重投入打法

同一层逻辑也适用于：

- 特殊弹药制造
- 武器与护甲维修
- 基地模块修复与升级

## 15. 钥匙卡与高价值区

### 15.1 钥匙卡来源

- 精英敌人掉落
- 高风险区事件终端奖励
- mini boss 掉落
- 钥匙卡碎片合成

### 15.2 钥匙卡作用

- 开启高价值区门禁
- 解锁深层武器箱或数据室
- 启用某些特殊撤离点

### 15.3 高价值区价值构成

高价值区的收益必须显著优于普通区域，优先掉落：

- 稀有模块
- 高级维修材料
- 穿甲弹蓝图或高级弹药材料
- 消音器核心零件
- 护甲核心
- 特殊蓝图
- 技能研究样本
- 成就相关物

## 16. 技能成长体系

### 16.1 技能树方向

MVP 建议只做 3 条主线：

- `机动系`：冲刺、闪避、位移强化、生存
- `火力系`：弱点、射击循环、爆发、弹药效率
- `回收系`：掉落质量、搜集效率、撤离收益、耐久管理

### 16.2 技能设计原则

- 以功能变化为主
- 少量数值强化为辅
- 必须支持不同玩法取向

### 16.3 技能点来源

- 研究样本
- 合同完成
- 成就解锁
- 首次进入高价值区
- 特定 boss 击杀

## 17. 成就与掉落解锁

成就系统不应只是展示页，而应承担内容解锁职责。

### 17.1 成就作用

- 解锁新掉落池
- 解锁新模组
- 解锁新合同
- 解锁新高风险事件
- 解锁新护甲词缀

### 17.2 成就示例

- 首次成功撤离
- 携带钥匙卡撤离
- 无伤击败 mini boss
- 单局进入并完成高价值区
- 使用某把武器完成特定击杀条件

## 18. 离线收益系统

### 18.1 目标

离线收益用于降低后期基础资源压力，提升回流，但不能替代主动游玩。

### 18.2 实现方式

通过 `无人机平台` 提供：

- 基础废料回收
- 低级数据采集
- 基础维修材料产出
- 少量低级基建材料回收

### 18.3 限制原则

- 只产出基础资源
- 不产出高级模块、钥匙卡、高价值战利品
- 累积时间有上限
- 需要基地升级才能提高效率

### 18.4 离线收益结算

根据玩家离线时间计算收益，并在下次登录时结算。

该系统应完全离线可运行，基于本地时间戳和存档状态结算。

## 19. 程序化内容策略

为了避免内容贫瘠，项目的可重复游玩性主要来自以下组合：

- 地图区块组合
- 区域风险组合
- 敌人词缀组合
- 模组组合
- 合同目标组合
- 高价值区奖励组合

程序化生成的重点不是完全随机，而是：

- `可控随机`
- `可读随机`
- `能形成决策的随机`

## 20. 信息与 UI 设计

### 20.1 局内 HUD

局内 HUD 只保留高价值信息：

- 血量
- 护甲状态
- 当前武器、当前弹种与弹药
- 武器耐久
- 快捷栏道具
- 背包容量压力
- 当前任务 / 撤离状态
- 钥匙卡状态
- 当前区域警戒或噪声压力
- 威胁等级

### 20.2 局外 UI

局外界面应重点服务：

- 配装
- 修复
- 升级
- 制造
- 技能选择
- 合同选择

避免过度装饰，保持清晰和功能性。

### 20.3 世界交互反馈

关键交互不能只依赖屏幕边缘按钮，必须在场景内给出明确反馈。

规则如下：

- 进入交互范围后，优先出现地面高亮、角框和几何箭头
- 底部提示条负责告诉玩家当前 `E` 会触发什么
- 出口、闸门这类关键节点应优先使用短读条，而不是瞬时切场
- 推进出口、基地出击闸门这类关键门体应尽量具备最小可见状态变化，例如开合、光栅退让或通道展开
- 地面掉落、出口 / 撤离点、区域核心、基地站点共用同一套亮色几何交互语言
- 若某个出口同时支持推进与撤离，`E` 默认执行主动作，次动作保留给附加 UI
- 撤离读条完成后应直接进入居中结算确认，不应再强制插入额外门体演出
- 读条、结算确认和门体开合若继续增强，其视觉语法仍必须沿用同一套亮色体系

这样可以保证：

- 玩家先在世界里看到交互，而不是先看 UI
- 画面风格不会因为功能增加而碎裂
- 后续新增站点、门禁和事件终端时，可以直接复用同一套原语

## 21. 保存与数据结构方向

存档需要支持：

- 局外基地状态
- 武器、护甲、模块库存
- 背包容量与仓库容量
- 技能树进度
- 成就进度
- 解锁池
- 离线收益时间戳
- 当前可选合同和区域状态

所有关键成长和解锁必须本地持久化。

## 22. MVP 成功标准

当以下条件同时满足时，MVP 即成立：

- 玩家能从基地准备并进入一局
- 地图足够大，存在多条探索路径
- 玩家能在局内体验搜集、战斗、背包取舍、耐久管理和撤离压力
- 玩家能感受到甲弹对抗、弹药取舍、怪物注意力和消音器带来的战术差异
- 玩家能获取钥匙卡并开启高价值区
- 玩家能把战利品带回基地并用于升级、修复、制造和技能成长
- 玩家能感受到下一局与上一局存在真实构筑差异
- 离线收益能减轻基础资源压力，但不破坏主动游玩价值

## 23. 后续扩展方向

在 MVP 打通后，再考虑扩展以下内容：

- 更多地图主题
- 更多怪物种类
- 更多高风险事件
- 更多 boss
- 更多武器平台
- 更多成就链
- 更复杂的高价值区规则
- 更复杂的区域环境危害

扩展顺序必须建立在 MVP 已经证明“撤离闭环 + 构筑闭环 + 局外成长闭环”成立的前提下。
