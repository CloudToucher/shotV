# 视觉与交互统一重构方案

## 1. 目标

这份文档只回答一件事：
后续所有画面和交互反馈，怎样在不偏离当前亮色几何语言的前提下继续推进。

核心原则：

- 不回退到局部暗色面板拼贴
- 不让常驻 UI 挤压场景主体
- 不把关键交互只做成右上角按钮
- 让 `角色 / 怪物 / 子弹 / 标记 / 面板` 说同一种视觉语言

## 2. 固定布局规则

### 2.1 战斗常态

- 左上：局部小地图
- 右上：仅在关键状态出现的行动坞站，例如推进、撤离、结算返回
- 右侧或上侧：不再保留长期霸屏的大侧栏
- 下中：条件出现的交互提示条
- HUD 本体：保留血量、武器、快捷栏、目标态势这类高价值信息

### 2.2 基地常态

- 左上：基地小地图
- 上中：当前待部署路线与基地标题
- 下中：条件出现的交互提示条
- 常态场景优先完整可见，仓储 / 工坊 / 指挥台都通过近距离交互进入

### 2.3 地图打开态

- 总地图必须全屏覆盖
- 打开总地图时，隐藏局部 HUD、交互条和行动坞站
- 地图上的图例、焦点目标和镜头框必须比场景常态更明确

### 2.4 终端面板态

- 仓储、工坊、指挥台、行动确认统一走全屏亮底面板
- 面板内部使用切角框、浅色底、结构线和小段强调色，不再使用厚重暗窗
- 面板关闭后必须立刻回到完整场景，而不是留下长期侧栏残影

## 3. 视觉令牌

### 3.1 底色

- 世界底色：冷白、浅青、轻灰
- 面板底色：高亮白底 + 低强度蓝色结构线
- 禁止重新引入大面积深灰或黑色主面板

### 3.2 语义色

- 友方 / 系统：`蓝`
- 行动 / 可操作：`橙`
- 危险 / 受击 / boss：`红`
- 撤离 / 通道联通：`亮橙偏黄`

### 3.3 结构

- 面板、格子、地图覆盖层统一使用切角多边形
- 高亮统一使用角框、细线、脉冲环，不用厚描边发光块
- 世界内反馈优先使用地面投影和几何括角，而不是悬浮大 UI

## 4. 世界交互原语

### 4.1 统一反馈链路

世界内交互统一采用三层反馈：

1. 地面高亮：脉冲环 + 角框 + 几何箭头
2. 底部提示条：说明当前 `E` 会做什么
3. 关键动作：近距离直接执行，必要时再配合右上角行动坞站

### 4.2 优先级

- 地面掉落
- 出口 / 撤离点
- 区域核心
- 基地站点

同一时间只强化当前最相关目标，避免屏幕同时闪多个点位。

### 4.3 当前已落地

- 基地站点：靠近后出现统一世界高亮，`E` 打开对应面板
- 战区出口：靠近后出现统一世界高亮，`E` 可进入推进 / 撤离读条
- 地面掉落：靠近后出现统一世界高亮，`E` 拾取并装入携行网格
- 基地出击闸门：靠近后出现统一世界高亮，`E` 先执行短暂校验，再打开部署确认
- 基地出击闸门与战区推进出口：已接入最小门体开合演示，保留 `校验 -> 开门 -> 执行` 的两段式状态
- 战区撤离：已改成 `校验 -> 结算确认`，读条结束后直接弹出居中结算面板，确认后回基地

### 4.4 仍待补完

- 真实门禁状态切换与碰撞变化
- 推进 / 出击侧更强的开门动画或传送平台反馈
- 结算面板与基地回顾之间的结果联动细化
- 更强的地图图例层和焦点层

## 5. 共享组件清单

### V0 已定

- 常态布局
- 地图覆盖态
- 全屏终端面板方向

### V1 已部分落地

- 亮色底
- 蓝 / 橙 / 红语义色
- 切角边框与结构线

### V2 已部分落地

- 全屏面板框
- 地图覆盖层
- 物品几何卡片
- 世界交互高亮

### V3 当前优先级

1. 战斗 HUD 收口
2. 基地常态 HUD 收口
3. 结算面板与出口反馈收口
4. 总地图图例层
5. 终端面板动效收口

## 6. 后续约束

- 新增功能如果需要 UI，先判断它属于 `常态信息`、`条件提示` 还是 `终端面板`
- 新增世界交互必须优先复用现有亮色几何高亮，不再各自发明一套提示
- 如果某个需求只能通过新增右侧栏才能完成，先回到这份文档重审布局，而不是直接落代码
